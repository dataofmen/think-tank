# Intake Skill — 지식 인테이크 4단계 검증

사용자가 기사, 링크, PDF, 이미지를 전달하면 4단계 비판적 검토를 실행하여 검증 통과한 인사이트만 토픽별 폴더에 저장합니다.

---

## Description

**트리거 조건**:
- 사용자가 `.inbox/` 또는 `.processing/`에 있는 pending 파일 처리 요청
- 기사 URL, PDF, 이미지가 포함된 메시지
- "분석해줘", "검증해줘", "intake" 등의 키워드

**자동 실행**: 대화 맥락에 따라 자동 발동됩니다.

---

## 4단계 검증 프로세스

### ① 출처 검증 (Source Verification)

**질문**: 이 정보는 어디서 왔는가?

**분류**:
- **[1차자료]**: 원저자가 직접 작성, 원본 연구 데이터, 공식 보고서
  - 예: 한국은행 보고서, 논문, 공식 통계
- **[2차해석]**: 누군가의 해석이나 요약
  - 예: 뉴스 기사, 블로그, 분석 글
- **[AI요약]**: AI가 요약/변환한 콘텐츠
  - 예: NotebookLM 결과, AI 번역

**출력**: 문서 상단에 태그 표시
```markdown
**출처**: [1차자료] / [2차해석] / [AI요약]
```

---

### ② 근거 검증 (Evidence Verification)

**질문**: 각 주장의 근거는 무엇인가?

**분류 기준**:
- 🟢 **사실**: 복수 소스에서 확인 가능, 데이터/통계 기반, 객관적
- 🟡 **해석**: 저자의 분석이나 의견이지만 근거가 명확함
- 🟠 **추정**: "~할 것으로 보인다", "~가능성이 있다", 조건부 주장
- 🔴 **의견**: 개인 주장, 근거 불명확, 주관적

**작업 방식**:
1. 본문을 주요 주장별로 분리
2. 각 주장마다 이모지 태그 부여
3. 근거 출처 명시 (데이터, 논문, 사례 등)

**출력 형식**:
```markdown
## 주요 주장 분석

1. 🟢 **주장 내용**
   - 근거: [출처 또는 데이터]
   
2. 🟡 **주장 내용**
   - 해석: [저자의 분석 근거]
   
3. 🔴 **주장 내용**
   - 의견: 근거 불명확
```

---

### ③ 반대 시나리오 (Counter-Scenario)

**질문**: 이 주장이 틀릴 수 있는 조건은?

**분석 관점**:
- **시장 가정**: 시장 상황이 바뀌면?
- **경쟁 구도**: 경쟁사가 다르게 움직이면?
- **사용자 행동**: 사용자가 예상과 다르게 반응하면?
- **기술 변화**: 기술 환경이 바뀌면?
- **규제/정책**: 법적 환경이 달라지면?

**작업 방식**:
1. 주요 주장 3-5개 선택
2. 각 주장마다 반박 가능한 조건 1-2개 작성
3. "그럼에도 유효한 이유" 함께 정리 (균형)

**출력 형식**:
```markdown
## 반대 시나리오

### 주장 1: [요약]
- **틀릴 수 있는 조건**: ...
- **그럼에도 유효한 이유**: ...

### 주장 2: [요약]
- **틀릴 수 있는 조건**: ...
- **그럼에도 유효한 이유**: ...
```

---

### ④ 적용성 분석 (Applicability)

**질문**: 이게 나한테 의미 있는가?

**판단 기준**:

1. **포트폴리오 연결**:
   - 진행 중인 프로젝트와 관련 있는가?
   - 과거 수행한 업무와 연결되는가?
   
2. **의사결정 영향**:
   - 현재 고민 중인 결정에 영향을 주는가?
   - 진행 중인 과제의 방향을 바꿀 만한가?
   
3. **액션 가능성**:
   - 당장 적용할 수 있는 방법론인가?
   - 다음 리서치에 쓸 수 있는가?

**적용성 점수**:
- **High**: 3가지 중 2개 이상 해당
- **Medium**: 3가지 중 1개 해당
- **Low**: 모두 해당 안 됨

**출력 형식**:
```markdown
## 적용성 분석

**적용성 점수**: High / Medium / Low

### 연결점
- 포트폴리오: [PRJ-006] 픽업 브랜드 충성도 리서치
- 의사결정: 정성 리서치 방법론 선택
- 액션: 다음 인터뷰 가이드 작성 시 이 프레임워크 적용 가능

### 구체적 활용 방안
- [어떻게 적용할 것인가]

### work-manager 연동 제안
- 연동 필요: Yes / No
- 대상 카드: [ID] 카드명
```

## 파일 저장 규칙

### 파일명 생성
```
YYYY-MM-DD-토픽-핵심키워드.md
```

예시:
- `2026-03-26-ux-research-progressive-disclosure.md`
- `2026-03-26-ai-productivity-cursor-workflow.md`
- `2026-03-26-product-strategy-growth-framework.md`

### 저장 위치
```
~/think-tank/intake/
```

토픽 폴더 구분 없이 `intake/` 루트에 직접 저장합니다. 토픽은 파일명과 메타데이터 태그로 구분합니다.

토픽 분류 (파일명 및 태그용):
- `ux-research`, `ai-productivity`, `product-strategy`
- `data-analysis`, `market-trends`, `business`, `tech`
- `uncategorized` (분류 불가 시)

### 원문 보존

외부 링크가 사라질 수 있으므로 크롤링한 원문을 별도 보관합니다:
```
~/think-tank/archive/YYYY-MM-DD-원본제목.md
```

archive 파일에는 원본 URL, 크롤링 일시, 원문 텍스트 전체를 저장합니다.

## 실행 절차

1. **입력 파일 확인**: `.inbox/pending-*.json` 또는 `.processing/processing-*.json`
2. **내용 읽기**: 타입에 따라 URL 크롤링 / PDF 추출 / 이미지 분석
3. **원문 보존**: `archive/YYYY-MM-DD-원본제목.md`에 원문 전체 저장
4. **4단계 검증 실행**
5. **토픽 자동 분류**: `topic-routing.md`의 키워드 매칭
6. **저장 여부 결정**: 상기 표 기준
7. **파일 저장**: `intake/YYYY-MM-DD-토픽-핵심키워드.md` (하위 폴더 없이 루트에 저장)
8. **work-manager 연동**: 적용성 분석에서 업무 연관 감지 시 제안
9. **처리 완료**: `.inbox/` 또는 `.processing/`에서 파일 이동 (`.processed/`로)
