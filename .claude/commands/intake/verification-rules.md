# 검증 규칙 상세 (Verification Rules)

4단계 검증의 구체적인 판단 기준과 실행 방법입니다.

---

## 1. 출처 검증 상세

### 1차 자료 판단 기준

**확실한 1차 자료**:
- 학술 논문 (peer-reviewed)
- 정부/공공기관 공식 보고서 (한국은행, 통계청, IMF, BIS 등)
- 기업 공식 IR 자료, 실적 발표
- 원저자가 직접 쓴 블로그 (본인 경험 기반)
- 원본 데이터셋, 로우 데이터

**2차 해석 판단 기준**:
- 뉴스 기사 (기자가 보도자료나 인터뷰 기반 작성)
- 분석 글, 해설 기사
- 블로그 리뷰 (다른 자료 기반 해석)
- 인용이 많은 글 (원본이 아님)

**AI 요약 판단 기준**:
- NotebookLM, ChatGPT, Claude 등 AI 도구가 생성
- "AI가 요약했다"는 명시
- 번역본 (원문과 의미 차이 가능)

### 출처 불명 시 처리
- 출처를 명확히 확인할 수 없으면 **[출처불명]** 태그
- 자동 저장하지 않고 사용자 확인 요청

## 2. 근거 검증 상세

### 각 등급별 구체적 기준

#### 🟢 사실 (Fact)
**조건 (모두 충족)**:
- 객관적으로 확인 가능
- 데이터/통계/수치로 뒷받침됨
- 복수 소스에서 일치하는 내용

**예시**:
- "2025년 배달앱 시장 규모는 27조원이다" (한국은행 데이터)
- "A사의 3분기 매출은 1,200억원이다" (공시 자료)

#### 🟡 해석 (Interpretation)
**조건**:
- 저자의 분석이나 의견
- 근거가 있지만 해석의 여지 존재
- 논리적 추론 과정이 명확함

**예시**:
- "이 데이터를 보면 사용자 이탈이 증가하는 추세다" (데이터는 사실, 추세 판단은 해석)
- "이 방법론이 효과적일 것으로 판단된다" (경험 기반 의견)

#### 🟠 추정 (Assumption)
**조건**:
- 조건부 주장 ("~라면", "~일 경우")
- 미래 예측
- "~로 보인다", "~가능성" 표현

**예시**:
- "향후 시장이 성장할 것으로 예상된다"
- "사용자들이 이 기능을 선호할 가능성이 높다"

#### 🔴 의견 (Opinion)
**조건**:
- 주관적 판단
- 근거 불명확
- 개인 선호나 믿음

**예시**:
- "이 디자인이 더 좋다"
- "이 방법이 최선이다" (대안 비교 없이)

### 혼합 문장 처리
한 문장에 여러 요소가 섞여 있으면 **가장 약한 등급**을 부여:
- "데이터를 보면(🟢) 이 트렌드가 계속될 것이다(🟠)" → 🟠 추정

## 3. 반대 시나리오 상세

### 반박 프레임워크

#### 3-1. 가정 역전
**원 주장**: "모바일 앱 사용자는 간편함을 선호한다"
**반대**: 일부 사용자(전문가, 파워유저)는 기능 풍부함을 더 선호할 수 있다

#### 3-2. 맥락 변경
**원 주장**: "이 마케팅 전략이 효과적이다"
**반대**: B2C에서 효과적이지만 B2B 환경에서는 작동하지 않을 수 있다

#### 3-3. 시간 변화
**원 주장**: "이 기술이 주류가 될 것이다"
**반대**: 단기적으로는 맞지만, 5년 후 더 나은 대안이 나올 수 있다

#### 3-4. 경쟁 반응
**원 주장**: "이 전략으로 시장 점유율을 높일 수 있다"
**반대**: 경쟁사가 빠르게 모방하면 차별화 우위가 사라진다

### 반박의 깊이
- **표면적 반박**: 단순 예외 케이스 지적
- **구조적 반박**: 가정 자체를 뒤집는 시나리오 (선호)

## 4. 적용성 분석 상세

### 포트폴리오 연결 체크리스트

진행 중인 프로젝트 목록은 `CONTEXT.md`의 **§3 포트폴리오** 섹션을 참조하세요.

**질문**:
- 이 인사이트가 진행 중인 프로젝트 중 하나와 관련 있는가?
- 과거 완료한 업무(achievements)와 연결되는가?

### 의사결정 영향 체크리스트

현재 진행 중인 의사결정 (`memory/decisions.md` 참조):
- (현재 없음)

**질문**:
- 이 인사이트가 의사결정에 영향을 주는가?
- 선택지의 우선순위를 바꾸는가?

### 액션 가능성 체크리스트

**질문**:
- 당장 다음 주에 적용할 수 있는가?
- 구체적인 행동으로 전환 가능한가?
- 누구에게, 어떻게 공유하면 유용한가?

적용성 분석에서 업무 연관성이 감지되면 work-manager 연동을 제안합니다.
연동 키워드, 제안 형식, 양방향 싱크 프로세스의 상세는 `link-to-work-manager.md`를 참조하세요.

## 6. 에러 처리

### 크롤링 실패
- 3회 재시도
- 실패 시 사용자에게 "본문 복사해서 보내주세요" 요청

### 검증 불가
- 정보가 너무 적거나 애매하면 **[검증불가]** 태그
- 사용자에게 "더 구체적인 자료가 필요합니다" 안내

### 토픽 분류 실패
- 어떤 토픽에도 매칭 안 되면 `intake/uncategorized/` 폴더에 임시 저장
- 사용자에게 "어느 토픽에 저장할까요?" 질문
