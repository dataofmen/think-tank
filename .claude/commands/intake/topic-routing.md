# 토픽 자동 분류 규칙 (Topic Routing)

인사이트를 7가지 토픽으로 자동 분류하는 키워드 매핑과 우선순위 규칙입니다.

---

## 토픽별 키워드 맵

### 1. ux-research (UX 리서치)

**핵심 키워드**:
- UX, UI, 사용자 경험, 유저빌리티
- 리서치, 연구, 조사, 인터뷰, 설문
- 페르소나, 사용자 여정, 고객 경험
- 정성 조사, 정량 조사
- A/B 테스트, 프로토타입, 와이어프레임

**매칭 점수**: 키워드 1개당 +5점

---

### 2. ai-productivity (AI 생산성)

**핵심 키워드**:
- AI, 인공지능, 머신러닝, LLM
- 생산성, 자동화, 워크플로우, 효율
- Claude, ChatGPT, GPT, Cursor, Copilot
- 프롬프트, 에이전트, RAG
- 노코드, 로우코드

**매칭 점수**: 키워드 1개당 +5점

---

### 3. product-strategy (프로덕트 전략)

**핵심 키워드**:

- 전략, 로드맵, 비전, 미션
- 그로스, 성장, 리텐션, 전환율
- PMF, 고객 세그먼트, 포지셔닝
- 마켓플레이스, 플랫폼, 이커머스

**매칭 점수**: 키워드 1개당 +5점

---

**매칭 점수**: 키워드 1개당 +5점

## 분류 알고리즘

### Step 1: 키워드 매칭

1. 제목과 본문에서 모든 키워드 검색
2. 각 토픽별 매칭 점수 계산
3. 가장 높은 점수의 토픽 선택

### Step 2: 우선순위 규칙

**복수 토픽 매칭 시** (동점 또는 근접):

1. **UX Research 우선**: 사용자 관련 키워드 있으면 UX Research
2. **AI Productivity 우선**: AI 도구/워크플로우 언급 시
3. **Product Strategy 우선**: 전략적 의사결정 맥락 시

**예시**:
- "AI를 활용한 UX 리서치" → ux-research (사용자 연구가 메인)
- "프로덕트 전략을 위한 데이터 분석" → product-strategy (전략이 목적)

### Step 3: 미분류 처리

점수가 10점 미만이면:
- `intake/uncategorized/` 폴더에 임시 저장
- 사용자에게 "어느 토픽에 저장할까요?" 질문
- 수동 분류 후 해당 토픽 키워드에 추가 학습

---

## 복합 토픽 처리

### 교차 토픽 인사이트

하나의 인사이트가 여러 토픽에 해당할 경우:

**방법 1: 주요 토픽에 저장 + 태그로 표시**
```markdown
**토픽**: ux-research
**관련 토픽**: #ai-productivity #data-analysis
```

**방법 2: 심링크 생성** (선택사항)
- 주요 토픽에 원본 저장
- 관련 토픽에 심링크 파일 생성 (내용: 원본 경로 링크)

## 키워드 학습 및 개선

### 자동 학습
- 사용자가 수동으로 토픽 변경하면:
  1. 해당 인사이트의 키워드 분석
  2. 새 키워드를 해당 토픽에 추가
  3. `topic-routing.md` 자동 업데이트

### 월별 리뷰
- 분류 오류 케이스 모음
- 키워드 맵 개선
- 우선순위 규칙 조정

---

## 특수 케이스

### 한국어 + 영어 혼용
- 둘 다 검색: "UX", "사용자 경험"
- 동의어 자동 매칭: "리서치" = "연구" = "조사"

### 약어 처리
- PM = 프로덕트 매니저 (product-strategy)
- KPI = 지표 (data-analysis)
- OKR = 목표 관리 (business)

### 맥락 기반 분류
단순 키워드 매칭으로 애매할 때:
- 제목과 첫 단락에 가중치 2배
- 저자 소속/배경 고려 (UX 리서처가 쓴 글 → ux-research 우선)
