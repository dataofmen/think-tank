# 핵심 워크플로우 (Workflows)

> Think Tank에 정보를 넣는 5가지 경로와 각각의 처리 방식입니다.

---

## 1. 기사/글 — 그냥 던짐

### 입력 방식
- 기사 URL을 Telegram에 전송
- 블로그, 뉴스, X 스레드 등

### 자동 처리
1. 본문 크롤링 (requests + BeautifulSoup)
2. 본문 내 이미지 URL 추출
3. 4단계 검증 실행
4. 토픽 자동 분류
5. `intake/[토픽]/` 저장

### 검증 기준
- ① 출처: 1차 자료 vs 2차 해석 구분
- ② 근거: 🟢사실 / 🟡해석 / 🟠추정 / 🔴의견
- ③ 반대 시나리오: "이게 틀릴 수 있는 조건은?"
- ④ 적용성: High/Medium/Low

---

## 2. PDF 리포트 — 원본을 직접 읽힘

### 입력 방식
- PDF 파일을 Telegram에 전송
- 또는 맥미니 `inbox-pdf/` 폴더에 직접 저장

### 자동 처리
1. 텍스트 추출 (Anthropic PDF Processing 스킬)
2. 챕터별 분리 (목차 기반)
3. 4단계 검증 실행
4. 저장

### 선호하는 1차 자료 소스
- 한국은행 보고서
- IMF 전망
- BIS 리포트
- Fed 발표
- a16z 크립토 리포트
- Messari 리서치

---

## 3. 유튜브 — NotebookLM 경유

### 입력 방식
1. NotebookLM에 YouTube URL 소스 추가
2. 프롬프트 입력:
   ```
   이 영상의 내용을 아래 구조로 요약해줘. 
   원저자의 주장을 왜곡하지 말고 있는 그대로 정리해줘.
   - 영상 정보 (제목, 채널명, 핵심 주제)
   - 주요 주장 최대 5개 (근거 없으면 '의견' 표기)
   - 인상적인 문장 원문 그대로 인용
   ```
3. NotebookLM 결과를 Telegram에 전송 (또는 복붙)

### 자동 처리
1. **[2차요약]** 태그 자동 추가
2. 4단계 검증 (원본 아님을 인식하고 정확성에 더 주의)
3. 저장

---

## 4. 책/독후감 — 프레임워크 추출

### 입력 방식
- 책 내용이나 독후감 텍스트 전송

### 자동 처리
1. 4단계 검증 실행
2. **추가 단계**:
   - 저자의 핵심 프레임워크 추출
   - 나한테 유용한 부분만 정리
   - "이 프레임워크로 내 상황을 보면?" 적용

---

## 5. 웹 리서치 — 반자동 (사용자 승인)

### 입력 방식
- "환율 전망 어때?" 같은 질문
- 에이전트가 직접 웹 검색

### 자동 처리
1. 복수 소스 검색 및 교차 검증
2. 확신도 표기:
   - 🟢 높음: 복수 소스 일치
   - 🟡 중간: 단일 소스
   - 🔴 낮음: AI 추정
3. **자동 저장 안 함** — 사용자가 "저장해둘까?" 승인 후 저장

---

## 워크플로우 요약

| 경로 | 입력 | 자동화 레벨 | 저장 조건 |
|------|------|-------------|----------|
| 기사/글 | URL | 완전 자동 | ① + ④ 통과 |
| PDF | 파일 | 완전 자동 | ① + ④ 통과 |
| 유튜브 | NotebookLM 결과 | 완전 자동 | [2차요약] + ④ 통과 |
| 책 | 텍스트 | 완전 자동 | ① + ④ 통과 + 프레임워크 |
| 웹 리서치 | 질문 | 반자동 | 사용자 승인 |
